## TASK A — REMOVE EVENTS MENU

### Step A1 — Sidebar Menu

Cari di `Sidebar.jsx` (atau `Navigation.jsx` / `BottomNav.jsx`) array yang mendefinisikan menu items. Pattern biasanya:

```javascript
const menuItems = [
  { id: 'home', label: 'HOME', icon: Home, path: '/' },
  { id: 'nutrition', label: 'NUTRITION', icon: Utensils, path: '/nutrition' },
  { id: 'progress', label: 'PROGRESS', icon: TrendingUp, path: '/progress' },
  { id: 'events', label: 'EVENTS', icon: Calendar, path: '/events' },  // ← REMOVE
  { id: 'moments', label: 'MOMENTS', icon: Camera, path: '/moments' },
  { id: 'profile', label: 'PROFILE', icon: User, path: '/profile' },
];
```

**Hapus** baris yang berisi `events` / `EVENTS` / `/events`.

Kalau ada `import { Calendar } from 'lucide-react'` yang **hanya dipakai untuk Events**, hapus juga. Tapi kalau Calendar masih dipakai di komponen lain (misal weather), **jangan hapus import-nya**.

### Step A2 — Remove Route

Di router (biasanya `App.jsx` atau `main.jsx`):

```jsx
<Route path="/events" element={<Events />} />        // ← REMOVE
import Events from './pages/Events';                  // ← REMOVE import
```

### Step A3 — Delete Events Page File

Hapus file `src/pages/Events.jsx`.

Kalau Replit Agent tidak bisa delete langsung, kosongkan isi file dengan komentar:
```jsx
// File deprecated — Events menu removed from app
// Safe to delete this file manually
```

### Step A4 — Cek Mobile Bottom Nav

Kalau ada file terpisah untuk bottom navigation di mobile (`BottomNav.jsx`), check juga — kemungkinan ada array menu yang sama. Pastikan mobile bottom nav menampilkan 5 tabs.
